How it works

A universal ingestion + retrieval pipeline designed for any domain, with progressive structure and explainable outcomes.

1) Ingest & normalize

Connect your mailbox (IMAP), sync manual or automatic, extract email body + attachments, and normalize into a canonical model.

  • Canonical entities: Document, Artifact, Chunk, Relations
  • Universal metadata: source, time, author, type, language
  • Deduplication: Message-ID + normalized fingerprint fallback

2) Structure progressively

Build lightweight hierarchy and summaries over time — without domain-specific assumptions.

  • Section/page hierarchy when available
  • Email ↔ attachment relationships
  • Summaries per section and per document

3) Index multi-representation

Multiple search views for higher recall: chunk, document, title/subject, and sparse tokens for rare identifiers.

  • Chunk embeddings + doc-level embeddings
  • Title/subject embeddings
  • Sparse representation for exact matches

4) Retrieve, rerank, cite

Candidate generation → reranking → consolidation → grounded response with citations (or explicit “not found”).

  • Hybrid candidate generation
  • Lightweight reranking for precision
  • Citations and confidence signals

FAQ

Short, direct answers.

Does Vixai work outside a specific niche?

Yes. The ingestion model and retrieval pipeline are domain-agnostic. Optional per-tenant calibration improves relevance without custom development.

Will Vixai hallucinate answers?

Grounding is mandatory. Responses include citations, and Vixai can explicitly return “not found” when evidence is insufficient.

Can I integrate via API only?

Yes. The same retrieval engine that powers the product can be exposed as an API layer for your application.

Do you support automatic sync?

Yes. Sync runs can be manual or scheduled, with job status, progress, and error visibility.
